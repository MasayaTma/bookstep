# AI読書記録アプリ MVP仕様書

## 1. 文書概要

### 1.1 文書名

AI読書記録アプリ MVP仕様書

### 1.2 アプリ仮称

BookStep

### 1.3 目的

ユーザーが読んだ本、読書時間、感想を記録し、読書状況を可視化する。

また、記録内容をもとにAIが簡単なコメントや次の行動を提案することで、読書を継続しやすくする。

### 1.4 MVPの基本方針

最初から多機能なSNSアプリを作るのではなく、以下の基本サイクルが動くことを優先する。

1. 本を登録する
2. 読書時間を記録する
3. 感想を入力する
4. 読書記録を一覧で確認する
5. AIからコメントを受け取る
6. 累計読書時間や継続状況を確認する

---

# 2. システムの目標

## 2.1 解決したい課題

読書を始めても、以下の理由で継続できないことがある。

* 読んだ記録が残らない
* どれくらい読んだか分からない
* 読書の成果を実感しにくい
* 読み終わった後に内容を振り返らない
* 次に何を読めばよいか分からない

本アプリでは、読書記録と簡単なフィードバックによって、ユーザーが読書の進捗や積み重ねを確認できるようにする。

## 2.2 MVPの成功条件

以下の操作が一通りエラーなく実行できること。

* 書籍を登録できる
* 登録した書籍を一覧表示できる
* 書籍の詳細を確認できる
* 読書時間を記録できる
* 感想を記録できる
* 累計読書時間を確認できる
* 読書日数を確認できる
* AIコメントを生成できる
* 登録内容を編集・削除できる

```text
     │       ├─ entity/
     │       ├─ dto/
     │       ├─ config/
     │       └─ BookStepApplication.java
     │
     └─ resources/
         ├─ templates/
         ├─ static/
         │   ├─ css/
         │   └─ js/
         └─ application.properties
```

## 7.2 読書記録機能

### FR-R001 読書記録登録

ユーザーは書籍ごとに読書時間と感想を登録できる。

### FR-R002 読書記録一覧

登録済みの読書記録を一覧表示できる。

読書時間、ページ、感想を編集できる。

登録済み読書記録を削除できる。

削除後は書籍の累計読書時間とポイントを再計算する。

### FR-R005 累計読書時間

書籍ごとに、関連する読書記録の読書時間を合計する。

### FR-R007 読書日数

読書記録が登録されている異なる日付の件数を表示する。

同じ日に複数の読書記録があっても、読書日数は1日として扱う。

# 8. AI機能

## 8.1 AIコメント生成

読書記録の内容をもとに、AIが短いコメントを生成する。

### 入力情報

* 書籍タイトル
* 著者名
* カテゴリ
* 読書時間
* 読んだページ
* ユーザーの感想
* これまでの累計読書時間

### 出力内容

以下を含む150文字から300文字程度の日本語コメントを生成する。

* 感想に対する肯定的なフィードバック
* 内容を深掘りするための質問
* 次回の読書で意識するポイント

### AIへの指示例

```text
あなたは読書習慣を支援するアシスタントです。

以下の読書記録に対して、日本語で150文字から300文字程度のコメントを生成してください。

コメントには以下を含めてください。

1. ユーザーの読書行動を肯定する
2. 感想の内容を深掘りする質問を1つ提示する
3. 次回の読書で意識するとよいポイントを1つ提示する

書籍タイトル：{bookTitle}
著者名：{author}
読書時間：{readingMinutes}分
読んだページ：{startPage}ページから{endPage}ページ
感想：{reflection}

過度に褒めすぎず、具体的で親しみやすい文章にしてください。
```

## 8.2 APIキーの扱い

APIキーはソースコードに直接記載しない。

環境変数から取得する。

```text
GEMINI_API_KEY
```

または以下を使用する。

```text
OPENAI_API_KEY
```

## 8.3 APIキーがない場合

APIキーが設定されていない場合もアプリを起動できるようにする。

AIコメント生成を実行した際は、以下の固定メッセージを表示する。

```text
AI APIキーが設定されていないため、コメントを生成できませんでした。
通常の読書記録は保存されています。
```

## 8.4 AI APIエラー時

AI APIの呼び出しに失敗しても、読書記録自体は保存する。

AI処理の失敗によって、書籍登録や読書記録登録をロールバックしない。

エラー内容を画面へ直接表示せず、ユーザー向けには以下を表示する。

```text
AIコメントの生成に失敗しました。時間を置いて再度お試しください。
```

# 9. ポイント機能

## 9.1 目的

読書の継続を実感できる簡易的なゲーム要素としてポイントを付与する。

金銭的価値は持たせない。

## 9.2 ポイントルール

* 読書記録の登録：10ポイント
* 30分以上の読書：追加で5ポイント
* 書籍を読了状態に変更：30ポイント
* 感想を50文字以上入力：追加で5ポイント

## 9.3 ポイント再計算

読書記録や書籍の状態を編集・削除した際に不整合が起きないよう、保存済みデータからポイントを再計算できる構成にする。

ポイントの数値を直接加算し続けるだけの実装にはしない。

# 13. 非機能要件

## 13.1 パフォーマンス

通常の画面表示は3秒以内を目標とする。

AI APIの応答待ちは最大30秒とし、タイムアウト時はエラーとして扱う。

## 13.2 セキュリティ

* APIキーをコードへ直接記載しない
* HTML出力時はエスケープ処理を行う
* 入力文字数を制限する
* SQLを文字列連結で作成しない
* 削除はPOSTリクエストで実行する
* CSRF対策を有効にする

ログイン機能を追加していないMVPでは、個人情報や機密情報を入力しない前提とする。

## 13.3 保守性

* Controller、Service、Repositoryを分離する
* Entityを画面の入力フォームとして直接使用しない
* 入力用DTOまたはFormクラスを作成する
* 定数やEnumを利用する
* 共通処理の重複を避ける
* 主要クラスに簡潔なJavaDocを記載する

## 13.4 エラーハンドリング

共通例外処理クラスを作成する。

```text
GlobalExceptionHandler
```

最低限、以下を処理する。

* 存在しないIDへのアクセス
* 入力値エラー
* データベースエラー
* AI APIエラー
* 予期しないエラー

# 14. テスト要件

## 14.1 Serviceテスト

最低限、以下をテストする。

* 書籍を登録できる
* タイトルが空の場合は登録できない
* 読書記録を登録できる
* 読書時間が0分の場合は登録できない
* 累計読書時間を正しく計算できる

# 16. 受け入れ条件

## AC-001 書籍登録

タイトルを入力して登録すると、書籍が保存され、詳細画面に表示される。

## AC-002 入力エラー

タイトルを空欄にして登録すると、エラーメッセージが表示され、データは保存されない。

## AC-003 読書記録

書籍、読書日、読書時間を入力すると、読書記録が保存される。

## AC-004 累計読書時間

同一書籍に30分と45分の記録を登録した場合、累計読書時間が75分と表示される。

## AC-005 読書日数

同じ日に2件の読書記録を登録した場合、読書日数は1日として表示される。

## AC-006 読了

書籍の状態を読了へ変更すると、読了書籍数が1冊増える。

## AC-007 AIコメント

APIキーが設定されている場合、読書記録をもとにAIコメントを生成できる。

## AC-008 AI障害

AI APIの呼び出しに失敗しても、読書記録は正常に保存される。

## AC-009 レスポンシブ表示

スマートフォン幅でも、主要なボタンと入力フォームを操作できる。

# 21. 開発上の注意

このアプリで最も重要なのは、多数の機能を実装することではなく、以下の一連の流れを完成させることである。

```text
書籍を登録する
↓
読書記録を登録する
↓
読書時間が集計される
↓
AIコメントが表示される
↓
ダッシュボードで積み重ねを確認する
```

MVPの段階では、デザインや高度なAI推薦よりも、データの登録・編集・削除・集計が正しく動作することを優先する。
